# XUIFAST v3 — Полная документация проекта

> Документ предназначен как для человека (разработчика / контрибьютора), так и для ИИ-ассистента, работающего с кодовой базой. Здесь описаны цели, архитектура, важные требования автора и технические детали.

---

## 1. Что такое XUIFAST

XUIFAST — это bash-скрипт «из коробки» для автоматической установки VPN-сервера на базе **3X-UI** + **Xray-core** с протоколом **VLESS** и технологиями маскировки **Reality** и **TLS**.

Целевая аудитория — **нетехнические пользователи**. Человек запускает одну команду на сервере, отвечает на пару вопросов и получает работающий VPN с готовыми ключами и QR-кодами.

**Репозиторий:** `https://github.com/anten-ka/self-signed-cert-script-by-antenka`  
**Ветки:** `main` (продакшн), `test` (разработка)  
**Автор:** Vitalii (anten-ka), YouTube-канал anten-ka  
**Язык интерфейса:** русский и английский (билингвальная i18n система)

---

## 2. Цели проекта

1. **Максимальная простота** — пользователю не нужно знать, что такое Xray, inbound, Reality. Скрипт всё делает сам.
2. **Красивый UX** — цветной вывод, Unicode-рамки, спиннеры, QR-коды прямо в терминале. Автор называет это «бархатный» (velvety) интерфейс.
3. **Два режима маскировки:**
   - **Lite (Reality)** — не нужен домен. Xray притворяется другим сайтом (yandex.ru, google.com и т.д.) через протокол Reality. DPI видит TLS-подключение к легитимному сайту.
   - **Pro (TLS)** — нужен домен. nginx + Let's Encrypt + реальный шаблон сайта + 3X-UI. DPI видит настоящий сайт с настоящим сертификатом.
4. **Панель остаётся доступной всегда** — даже если автонастройка VPN не удалась, 3X-UI установлена и доступна для ручной конфигурации. Скрипт — это «быстрый старт», а не единственный путь.
5. **Поддержка нескольких версий 3X-UI:**
   - **New Generation (3.x)** — Vue 3, новый API, современный UI
   - **Legacy (2.x)** — классический интерфейс, проверенная стабильность
6. **Выбор транспорта:** TCP (рекомендуется), XHTTP, gRPC
7. **Генерация нескольких VPN-ключей** — пользователь выбирает от 1 до 100 ключей при установке (по умолчанию 3)

## 3. Архитектура и структура файлов

### 3.1 Точка входа

```
bash <(curl -sL URL/bootstrap.sh)
```

`bootstrap.sh` — одноразовый загрузчик. Клонирует репозиторий в `/opt/xuifast-installer/` (ветка `test`), затем запускает `xuifast.sh`.

### 3.2 Главный скрипт — `xuifast.sh`

Содержит:
- `main()` — точка входа: выбор языка → проверка диска → если уже установлен — меню, иначе — установка
- `select_and_install()` — выбор режима (Lite / Pro)
- `install_lite()` — полный flow установки Reality
- `install_pro()` — полный flow установки TLS
- `post_install_flow()` — показ QR, тест подключения, список пользователей
- `main_menu()` + субменю — интерактивное управление после установки (команда `xuifast`)

### 3.3 Библиотеки (`lib/`)

| Файл | Назначение |
|------|-----------|
| `common.sh` | Версия, пути, цвета, логирование, спиннер, установка зависимостей, определение IP/geo, JSON-конфиг, работа с портами |
| `i18n.sh` | Движок интернационализации: `t()` для перевода, `tf()` для перевода с printf-форматированием |
| `lang/en.sh` | Английские строки (ассоциативный массив `I18N`) |
| `lang/ru.sh` | Русские строки |
| `xui.sh` | Установка/удаление 3X-UI, управление сервисом, извлечение/сохранение/загрузка credentials, генерация Reality-ключей |
| `xui_api.sh` | REST API 3X-UI: логин, CSRF, создание inbounds (Reality/TLS), генерация VLESS-ссылок, управление пользователями, показ credentials |
| `reality_domains.sh` | Списки доменов для маскировки: 50 российских + 50 международных, все с TLSv1.3 + H2 |
| `website.sh` | nginx: установка, конфигурация, сайт-шаблоны, SSL через certbot, авто-продление |
| `templates_catalog.sh` | Каталог ~1800 HTML-шаблонов: навигация, предпросмотр, git sparse-checkout, кастомные Git URL |

### 3.4 Пути на сервере (после установки)

| Путь | Что |
|------|-----|
| `/opt/xuifast-installer/` | Скрипт и модули (клон git) |
| `/opt/xuifast/` | Данные: `config.json`, `.language`, `templates_catalog.json` |
| `/opt/xuifast/config.json` | JSON-конфиг с режимом, IP, доменом, транспортом, версией |
| `/root/.xuifast_credentials` | Логин/пароль/URL панели (chmod 600) |
| `/usr/local/x-ui/` | Бинарники 3X-UI и Xray |
| `/etc/x-ui/x-ui.db` | SQLite база 3X-UI (пользователи, настройки, inbounds) |
| `/var/www/html/` | Корень веб-сайта (Pro mode) |
| `/etc/nginx/sites-available/xuifast` | nginx конфиг сайта |
| `/etc/letsencrypt/` | SSL-сертификаты Let's Encrypt |

## 4. Flow установки (пошагово)

### 4.1 Lite mode (Reality)

1. Выбор языка (EN/RU)
2. Выбор режима → Lite
3. Определение IP сервера + геолокации
4. Выбор домена для маскировки (из списка RU/INT, проверка TLSv1.3+H2)
5. Выбор версии 3X-UI (New Gen 3.x / Legacy 2.x)
6. Выбор транспорта (TCP / XHTTP / gRPC)
7. Количество VPN-ключей (1–100, по умолчанию 3)
8. Сводка конфигурации → подтверждение
9. Установка зависимостей (curl, jq, sqlite3, qrencode и т.д.)
10. Установка 3X-UI → извлечение credentials → сохранение
11. **Best-effort автонастройка:** API логин → генерация Reality keypair → создание пользователей → создание inbound на порт 443
12. Генерация VLESS-ссылок и QR-кодов
13. Показ credentials + предупреждение о сертификатах
14. Post-install: скачивание приложения, QR первого пользователя, тест подключения

### 4.2 Pro mode (TLS)

1–3. Аналогично Lite
4. Ввод домена (проверка DNS — домен должен указывать на IP сервера)
5. Email для SSL (опционально)
6. Выбор шаблона сайта (каталог ~1800 шаблонов / кастомный Git URL / стандартная заглушка)
7. Количество ключей, версия 3X-UI, транспорт
8. Сводка → подтверждение
9. Зависимости → освобождение порта 443
10. Развёртывание сайта: nginx → certbot → шаблон → SSL → конфигурация nginx
11. Установка 3X-UI → credentials
12. **Best-effort:** API → создание пользователей → TLS inbound с Let's Encrypt сертификатом
13. VLESS-ссылки → credentials → post-install

## 5. Ключевые технические решения

### 5.1 `set -euo pipefail` и grep

Скрипт использует строгий режим bash. Команды `grep` в пайплайне возвращают exit code 1 при отсутствии совпадений, что убивает скрипт. **Решение:** все `grep` в пайплайнах завершаются `|| true`.

Примеры:
```bash
port443_proc=$(ss -tlnp 'sport = :443' 2>/dev/null | grep -o 'users:(("[^"]*' | ... | head -1) || true
pids=$(ss -tlnp "sport = :${port}" 2>/dev/null | grep -o 'pid=[0-9]*' | ... | sort -u) || true
```

### 5.2 Best-effort автонастройка

Панель 3X-UI ВСЕГДА устанавливается и доступна. Автоматическое создание VPN-подключения (inbound, пользователи, VLESS-ссылки) — это **бонус**, а не обязательный шаг. Если API недоступен или создание inbound не удалось, пользователь получает credentials и может настроить вручную.

### 5.3 SSL и Let's Encrypt

- Certbot с retry: первая попытка → ожидание 5 сек → restart nginx → вторая попытка
- **Важно:** частые запросы сертификатов (при тестировании) приводят к **временному бану** Let's Encrypt (ошибка 403). При тестировании нужно сохранять существующие сертификаты между прогонами.

### 5.4 Сертификаты панели

10-летние self-signed сертификаты **больше не выпускаются**. Панель 3X-UI автоматически генерирует сертификат на 6 дней и автоматически продлевает его. Пользователю ничего делать не нужно — скрипт показывает об этом информацию после установки.

### 5.5 IPv4 в credentials

Credentials (логин/пароль/URL) показываются и сохраняются с **IPv4** адресом сервера. URL формата: `https://IP:PORT/PATH`.

### 5.6 Архитектура i18n

Все строки интерфейса вынесены в ассоциативный массив `I18N[]`. Функции: `t key` — получить перевод, `tf key args...` — перевод с printf-подстановкой. Языковые файлы — обычные bash-скрипты с присвоениями `I18N[key]="value"`.

### 5.7 Установка 3X-UI

Установка выполняется через официальный скрипт 3X-UI (`install.sh`), но **без expect**. Скрипт подставляет переменные окружения или патчит конфиг после установки. Поддерживаются две ветки:
- `main` — New Generation (3.x)
- `master` (или `main` для старых версий) — Legacy (2.x)

## 6. Важные требования автора

> Эти замечания собраны из всей истории разработки. ИИ-ассистенту ОБЯЗАТЕЛЬНО их учитывать.

### 6.1 UX для нетехнических пользователей
- Скрипт должен быть **foolproof** — защита от дурака на каждом шаге
- Красивый вывод: цвета, юникод, спиннеры, но **без рамок-боксов** (`╔═╗║╚╝`) — они разъезжаются с эмодзи. Вместо них — простые горизонтальные линии `═══`
- Все сообщения об ошибках — человекопонятные, на выбранном языке

### 6.2 Панель всегда доступна
- Даже если автонастройка VPN не удалась, **панель установлена и работает**
- Скрипт — это «быстрый старт», а не единственный способ настройки
- Credentials всегда показываются и сохраняются в файл

### 6.3 Credentials после установки
- Должны отображаться в красивой рамке из `═══` (не боксы)
- Поля: логин, пароль, URL
- Предупреждение: «ОБЯЗАТЕЛЬНО СОХРАНИТЕ ЭТИ ДАННЫЕ!»
- Информация о сертификатах: 10-летние не выпускаются, панель сама генерит на 6 дней
- Используется **IPv4** адрес (не IPv6)

### 6.4 Тестирование
- Автор ценит **тщательное тестирование** перед релизом
- Тестируются все 4 комбинации: Lite×New Gen, Lite×Legacy, Pro×New Gen, Pro×Legacy
- При тестировании Pro mode учитывать rate limits Let's Encrypt
- Паузы между проверками: **10–15 секунд**, не 3–4 минуты

### 6.5 Git workflow
- Разработка ведётся в ветке `test`
- Пуш в `main` — **только по явной просьбе** автора
- При пуше в `main` **всегда создавать GitHub Release** с повышением версии
- PAT для GitHub хранится в CLAUDE.md (не публиковать)

### 6.6 Количество ключей
- Пользователь выбирает количество VPN-ключей при установке (1–100)
- По умолчанию — 3
- Валидация ввода: если не число или вне диапазона → 3

## 8. API 3X-UI

Скрипт взаимодействует с REST API панели:

| Действие | Эндпоинт | Метод |
|----------|----------|-------|
| Логин | `/login` | POST (username, password) |
| Получить inbounds | `/panel/api/inbounds/list` | GET |
| Создать inbound | `/panel/api/inbounds/add` | POST (JSON) |
| Проверить онлайн | `/panel/api/inbounds/onlines` | POST |
| Установить язык | `/panel/setting/update` | POST |

CSRF-токен: берётся из cookie после логина, передаётся в заголовке `X-Csrf-Token` или параметре.

## 9. Зависимости

Скрипт автоматически устанавливает через `apt`:
- `curl`, `wget` — загрузка
- `jq` — парсинг JSON
- `sqlite3` — чтение БД 3X-UI
- `qrencode` — QR-коды в терминале
- `certbot`, `python3-certbot-nginx` — SSL (Pro mode)
- `nginx` — веб-сервер (Pro mode)
- `git` — шаблоны
- `openssl` — генерация паролей
- `python3` — вспомогательные скрипты (bcrypt, JSON-обработка)

## 10. Поддерживаемые ОС и архитектуры

- **ОС:** Ubuntu/Debian (apt-based)
- **Архитектуры:** amd64 (x86_64), arm64 (aarch64) — скрипт автоопределяет

## 11. Команды после установки

После установки скрипт создаёт алиас `xuifast`, который открывает интерактивное меню:

- **VPN** → установка/обновление, показ ссылок, QR, рестарт, логи, смена режима
- **Управление** → язык, рестарт, удаление
- **О программе** → версия, автор, credits

Меню обновляется каждые 30 секунд (показывает статус сервисов: 3X-UI, Xray, nginx).

## 12. Известные ограничения и edge cases

1. **Let's Encrypt rate limits** — при частом тестировании Pro mode можно получить временный бан. Решение: сохранять сертификаты между тестами.
2. **DNS пропагация** — Pro mode требует, чтобы домен указывал на IP сервера. DNS может обновляться до 48 часов.
3. **Порт 443** — должен быть свободен. Скрипт пытается освободить его автоматически (кроме xray).
4. **Шаблоны** — каталог templates_catalog.json содержит ~1800 шаблонов, но не все могут быть доступны (репозитории удаляются).
5. **Legacy 3X-UI** — некоторые API endpoint'ы отличаются между 2.x и 3.x. Скрипт обрабатывает обе версии.

## 13. Для ИИ-ассистента: чеклист при работе с кодом

При внесении изменений в кодовую базу XUIFAST:

1. **Все grep в пайплайнах** → добавлять `|| true` (из-за `set -euo pipefail`)
2. **Строки UI** → выносить в `I18N[]` в ОБА файла (en.sh + ru.sh)
3. **Рамки/боксы** → использовать только `═══` горизонтальные линии, никогда `╔═╗║╚╝`
4. **Credentials** → всегда IPv4 (get_server_ip), показывать URL с портом и путём
5. **Best-effort** → автонастройка VPN не должна ронять установку. Панель должна быть доступна всегда.
6. **Тесты** → 4 комбинации: Lite×NewGen, Lite×Legacy, Pro×NewGen, Pro×Legacy
7. **Git** → коммиты в `test`, в `main` только по команде, с Release и повышением версии
8. **Синтаксис** → `bash -n` на все .sh файлы перед коммитом
9. **Версия** → обновлять `XUIFAST_VERSION` в `lib/common.sh` при значимых изменениях
10. **Пароли/ключи** → не хардкодить, генерировать через `openssl rand`
